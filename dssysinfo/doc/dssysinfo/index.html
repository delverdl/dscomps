DSSysInfo library API

This is the system information library API, is has been tested in Windows 10 and Ubuntu Linux 18.04. User only need to use one header file to completely have access to library's functionality; CSysInfo class is the one exported to have access to both devices connector and system information interfaces; and thus access a random buffer filling method. You cannot have multiple instances of this class; it's only allowed one per process, and it's explained in the following code snippet

* # In PRO file
*
* CONFIG += dssysinfo
*
*
#include <CSysInfo>
//...
CSysInfo* m_ksysinfo = CSysInfo::instance();
//...

There's a good tutorial for using UDEV. and you should look at the Win32 WMI classes to know about how to query them and their properties. Most of windows CIM classes are provided in Windows desktop apps -> System Administration -> WMI Providers -> CIMWin32 WMI Providers -> Win32 Provider web page.
